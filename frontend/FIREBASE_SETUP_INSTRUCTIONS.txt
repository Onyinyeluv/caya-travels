STEPS TO COMPLETE:
============================================

1. ENABLE FIRESTORE DATABASE
   ----------------------------
   a. Go to: https://console.firebase.google.com/
   b. Select your "Caya Express Travel" project
   c. Click "Firestore Database" in left sidebar
   d. Click "Create database"
   e. Select "Start in production mode"
   f. Choose location: "us-central1" (or closest to Nigeria)
   g. Click "Enable" and wait ~30 seconds

2. SET UP FIRESTORE RULES
   ----------------------------
   a. After database is created, click "Rules" tab
   b. Replace ALL the text with this:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

   c. Click "Publish"

3. ENABLE AUTHENTICATION
   ----------------------------
   a. Click "Authentication" in left sidebar
   b. Click "Get started"
   c. Click "Email/Password"
   d. Toggle the switch to "Enable" ✅
   e. Click "Save"

4. ADD ADMIN USER
   ----------------------------
   a. Still in "Authentication", click "Users" tab
   b. Click "Add user"
   c. Email: cayaexpress travels@gmail.com (or your preferred email)
   d. Password: Create a STRONG password (remember it!)
   e. Click "Add user"

5. UPLOAD FILES TO YOUR HOSTING
   ----------------------------
   Upload these files to your hosting cPanel:
   
   ✅ admin-partners-firebase.html (NEW file)
   ✅ partner.html (UPDATED file)

6. ACCESS YOUR NEW ADMIN PORTAL
   ----------------------------
   URL: https://cayaexpresstravel.com/admin-partners-firebase.html
   
   Login with the email and password you created in Step 4

============================================
TROUBLESHOOTING:
============================================

❌ "Firebase not initialized"
   → Wait 2-3 seconds after page loads, refresh if needed

❌ "Permission denied"
   → Make sure you completed Step 2 (Firestore Rules)

❌ "Invalid email or password"
   → Use the exact email/password from Step 4

❌ "No partners showing"
   → Test by submitting the partner form first
